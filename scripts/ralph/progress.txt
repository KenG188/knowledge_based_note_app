## Codebase Patterns
- vitest.config MUST use .mts extension — vite 7 dropped CJS support, so vitest.config.ts fails with ERR_REQUIRE_ESM. Always use vitest.config.mts
- Next.js App Router dynamic route params are now a Promise: `const { id } = await params`
- SQLite tags stored as JSON string, parsed with JSON.parse on read
- Import alias `@/` maps to `src/` directory
- No "type": "module" in package.json — project uses CommonJS default
- Package manager: npm
- Test command: npx vitest run (or npm test after adding script)
- Typecheck command: npx tsc --noEmit
- DB singleton requires closeDb() call in test afterEach before deleting DB file, otherwise old connection persists and data leaks between tests
